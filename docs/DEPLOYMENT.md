## Part 1 вҖ” Supabase database

### 1. Create a project

1. Sign in at [supabase.com](https://supabase.com) вҶ’ **New project**
2. Choose a region close to your users (e.g. **Europe / Frankfurt**)
3. Save the database password

### 2. Copy the connection string

**Settings вҶ’ Database вҶ’ Connection string вҶ’ URI** вҖ” use the **Transaction pooler** (port **6543**):

```text
postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:6543/postgres?sslmode=require
```

URL-encode special characters in the password (`@` вҶ’ `%40`, etc.).

## Part 2 вҖ” Render backend

### Option A вҖ” Blueprint (recommended)

The repo includes [`render.yaml`](../render.yaml). On Render:

1. **New вҶ’ Blueprint** вҶ’ connect your GitHub repo
2. Render creates the `ai-job-finder-api` web service from the blueprint
3. When prompted, set these **secret** env vars:

| Variable | Value |
|----------|--------|
| `DATABASE_URL` | Supabase pooler URI (port 6543) |
| `CORS_ORIGINS` | Your Vercel production URL, e.g. `https://ai-job-finder.vercel.app` |

The blueprint already sets:

- `APP_ENV=production`
- `DB_USE_NULL_POOL=true`
- `CORS_ORIGIN_REGEX=https://.*\.vercel\.app` (Vercel preview deployments)
- `JOB_FETCH_ON_STARTUP=false` (faster cold starts on free tier)
- `JWT_SECRET` вҖ” auto-generated by Render

### Option B вҖ” Manual web service

1. **New вҶ’ Web Service** вҶ’ connect repo
2. Settings:

| Setting | Value |
|---------|--------|
| **Root Directory** | `backend` |
| **Runtime** | Python 3 |
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `uvicorn app.main:app --host 0.0.0.0 --port $PORT` |
| **Health Check Path** | `/health` |

3. Add environment variables (see [Environment variables](#environment-variables-reference))

### 3. Run database migrations (once)

Before the API can serve traffic, create PostgreSQL tables:

1. Render dashboard вҶ’ your service вҶ’ **Shell**
2. Run:

```bash
python scripts/db_check.py
python scripts/db_init.py
```

Confirm tables in Supabase **Table Editor** (`users`, `jobs`, `cv_profiles`, `job_matches`, вҖҰ).

### 4. Verify backend

```text
GET https://your-backend.onrender.com/health
```

Expected:

```json
{
  "status": "ok",
  "environment": "production",
  "database": "postgresql_connected",
  "database_type": "postgresql"
}
```

> **Free tier note:** Render spins down after inactivity. The first request may take ~30s (cold start).

## Part 3 вҖ” Vercel frontend

### 1. Import project

1. [vercel.com/new](https://vercel.com/new) вҶ’ import your GitHub repo
2. Configure:

| Setting | Value |
|---------|--------|
| **Root Directory** | `frontend` |
| **Framework Preset** | Vite |
| **Build Command** | `npm run build` |
| **Output Directory** | `dist` |

The repo includes [`frontend/vercel.json`](../frontend/vercel.json) for SPA routing (React Router).

### 2. Environment variables

In **Project Settings вҶ’ Environment Variables**:

| Variable | Environment | Value |
|----------|-------------|--------|
| `VITE_API_URL` | Production | `https://your-backend.onrender.com` (no trailing slash) |
| `VITE_USE_MOCK` | Production | `false` |

Template: [`frontend/.env.production.example`](../frontend/.env.production.example)

For **Preview** deployments, set the same `VITE_API_URL` unless you use a staging backend.

### 3. Deploy

Click **Deploy**. Vercel builds `frontend/dist` and serves it globally.

### 4. Update backend CORS

After Vercel gives you a URL (e.g. `https://ai-job-finder.vercel.app`), set on **Render**:

```env
CORS_ORIGINS=https://ai-job-finder.vercel.app
```

Redeploy the backend if it was deployed before CORS was configured.

## Part 4 вҖ” CORS between Vercel and Render

The backend reads CORS from environment variables:

| Variable | Purpose |
|----------|---------|
| `CORS_ORIGINS` | Comma-separated exact origins (production domain) |
| `CORS_ORIGIN_REGEX` | Optional regex for Vercel preview URLs |

**Production example (Render env vars):**

```env
CORS_ORIGINS=https://ai-job-finder.vercel.app,https://www.yourdomain.com
CORS_ORIGIN_REGEX=https://.*\.vercel\.app
```

**How it works:**

- Browser requests from `https://ai-job-finder.vercel.app` вҶ’ allowed via `CORS_ORIGINS`
- Preview deploys like `https://ai-job-finder-git-main-user.vercel.app` вҶ’ allowed via `CORS_ORIGIN_REGEX`
- Credentials (JWT cookies/headers) are enabled (`allow_credentials=True`)

**Test CORS from browser console** (on your Vercel site):

```javascript
fetch('https://your-backend.onrender.com/health').then(r => r.json()).then(console.log)
```

If CORS fails, check Render logs and confirm `CORS_ORIGINS` matches your Vercel URL exactly (including `https://`, no trailing slash).

## Troubleshooting

### `APP_ENV=production cannot use SQLite`

Set `DATABASE_URL` to Supabase on Render вҖ” not the local SQLite path.

### `JWT_SECRET must be a random string`

Generate one: `python -c "import secrets; print(secrets.token_urlsafe(48))"` and set on Render.

### CORS error in browser

- Add your exact Vercel URL to `CORS_ORIGINS`
- Include `https://` prefix, no trailing slash
- Redeploy backend after changing env vars

### `SSL connection required`

Add `?sslmode=require` to `DATABASE_URL` (auto-added for `.supabase.co` hosts).

### Migrations fail on pooler

Run `python scripts/db_init.py` вҖ” it tries session pooler (5432) before transaction pooler (6543). Or set `MIGRATION_DATABASE_URL` to the session pooler URI.

### API hangs on startup

Ensure `DB_RUN_MIGRATIONS_ON_STARTUP=false` (default). Run migrations manually via Shell.

### Frontend shows mock data

Set `VITE_USE_MOCK=false` on Vercel and redeploy.

### Frontend cannot reach API

Confirm `VITE_API_URL` is set in Vercel **Production** environment and rebuild.
